curl https://sh.rustup.rs -sSf | sh

关于 Rust 安装的注意事项

使用 rustup 管理工具链

Rust 由 rustup 工具来安装和管理。 Rust 有一个6周的 快速发布过程 并且支持 大量的平台 ，所以任何时候都有很多 Rust 构建可用。 rustup manages these builds in a consistent way on every platform that Rust supports, enabling installation of Rust from the beta and nightly release channels as well as support for additional cross-compilation targets

For more information see the rustup documentation.

配置 PATH 环境变量

在 Rust 开发环境中，所有工具都安装到 ~/.cargo/bin %USERPROFILE%\.cargo\bin 目录， 并且您能够在这里找到 Rust 工具链，包括 rustc、cargo及rustup。

Accordingly, it is customary for Rust developers to include this directory in their PATH environment variable. During installation, rustup, will attempt to configure PATH, but because of differences between platforms, command shells, and bugs in rustup, the modifications to PATH may not take effect until the console is restarted, or the user logged out, or may not succeed at all.

当安装完成之后，如果在控制台运行 rustc --version 失败，这是最可能的原因。

Windows 注意事项

在 Windows 上， Rust 需要 Visual C++ 生成工具 2013 或更新版本的支持。获取 Visual C++ 生成工具最方便的方法是安装 Microsoft Visual C++ Build Tools 2015 。或者，你可以 安装 Visual Studio 2015 或 Visual Studio 2013 并在安装过程中选择安装 “C++ 工具”。

关于在 Windows 上配置 Rust 的更多信息请查看 Windows-specific rustup 文档。

其他安装方法

上述通过 rustup 的安装方法是大多数开发者的首选。 但是 Rust 也可以 通过其他方法安装。
